# Repository Guidelines for knowledge/

## 目录定位
`knowledge/` 用于沉淀个人学习和 AGENT 协作过程中形成的技术知识，主要服务于后续复习、面试准备和知识体系梳理。

本目录关注“知识理解与表达”，而不是项目功能如何实现。

## 内容范围
本目录适合记录：
- Java、后端、数据库、分布式、AI 应用等技术知识点。
- 用户学习后与 AGENT 讨论、校准、补充形成的总结。
- 面试中可能被问到的概念、原理、场景和追问。
- 从项目开发过程中提炼出的通用经验和可讲述素材。
- 当前知识点可以继续深入或横向扩展的关联方向。

本目录不用于记录：
- 具体业务需求。
- 项目接口文档。
- 临时开发计划。
- 构建、测试、部署命令。
- 代码实现规范。

## 协作方式
用户通常会先自行了解某个知识点，再与 AGENT 讨论理解、边界、误区和关联问题。讨论收敛后，AGENT 可以帮助将内容整理成适合复习和面试表达的 Markdown 文档。

整理时应优先保证内容准确、结构清楚、方便回看，不强制套用固定模板。

当用户说“结合当前项目”“基于当前项目”“用当前项目做测试/demo”等类似表述时，默认“当前项目”固定指 `algo-mentor` 仓库，即用户语境中的 `/code/algo-mentor` 项目，而不是其他临时目录、示例仓库或外部项目。知识整理、案例说明、测试或 demo 如需依托项目背景，也应优先使用 `/code/algo-mentor`。

## 写作原则
- 默认使用中文。
- 文档名称默认使用中文，文件名应能直接表达主题，便于复习和检索。
- 结论尽量明确，避免空泛描述。
- 保留必要背景，但不要写成流水账。
- 对容易混淆或容易被追问的点，可以适当展开。
- 文档末尾可以补充相关联的深入方向或横向扩展主题，方便后续继续学习。
- 具体工程实现规范仍以根目录 `AGENTS.md` 或对应模块说明为准。

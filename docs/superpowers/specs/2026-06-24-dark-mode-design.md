# 暗色模式设计

## 背景

当前前端已经具备统一应用外壳、登录页、题库、学习计划和 AI 调试页面。全局样式集中在 `frontend/src/styles.css`，并且已经大量使用 CSS 变量，例如页面背景、卡片背景、边框、正文颜色、主按钮颜色和状态色。

截图中的 demo 在右上角提供月亮按钮作为暗色模式入口。用户确认第一版采用简单开关：默认浅色，不跟随系统主题；暗色视觉方向采用中性深色，保留黄色品牌动作色。

## 目标

- 提供浅色和暗色两种主题。
- 默认使用浅色主题。
- 用户在登录后的顶部导航右侧通过图标按钮切换主题。
- 切换结果写入 `localStorage`，刷新后保持用户选择。
- 已保存主题全站生效，包括登录页、加载态、题库、学习计划和 AI 调试页面。
- 暗色视觉采用中性深色：深蓝灰页面背景、较浅的卡片层、柔和边框、清晰文本层级、保留黄色主按钮。
- 保持现有页面结构、数据流和 API 不变。

## 非目标

- 不跟随 `prefers-color-scheme`。
- 不提供“跟随系统 / 浅色 / 暗色”的三态设置。
- 不在登录页新增主题切换按钮。
- 不引入主题库、CSS-in-JS 或额外 UI 依赖。
- 不重做页面布局、导航结构或组件信息架构。
- 不新增后端接口或数据库字段保存主题偏好。

## 用户体验

登录后的应用顶部右侧增加一个图标按钮，位置放在语言选择器附近。当前是浅色主题时，按钮显示月亮图标，点击后切换为暗色；当前是暗色主题时，按钮显示太阳图标，点击后切换为浅色。

按钮需要提供可访问名称。中文环境下可使用“切换为深色模式”和“切换为浅色模式”，英文环境下使用对应英文文案。

登录页不展示切换入口，但会读取已保存主题。因此用户如果在登录后切到暗色，再刷新或退出到登录页，登录页仍使用暗色变量。

## 主题状态模型

主题只支持两个值：

- `light`
- `dark`

本地存储 key 使用 `algo-mentor-theme`。读取时只接受上述两个值，其他值视为无效并回退到 `light`。

应用启动时读取本地存储，确定初始主题。主题变化时同步执行两件事：

- 写入 `document.documentElement.dataset.theme`。
- 写入 `localStorage`。

浅色主题可以不设置 `data-theme`，也可以设置为 `light`。为方便测试和行为一致，第一版建议显式设置 `data-theme="light"` 或 `data-theme="dark"`。

如果 `localStorage` 不可用，应用仍保持可用：默认浅色，点击按钮可在当前页面生命周期内切换，但忽略持久化失败。

## 技术设计

### 主题边界

新增一个轻量主题模块，负责主题值、存储 key、读取、写入和 DOM 应用。该模块不依赖业务组件，便于单测。

建议职责：

- 校验主题值。
- 从 `localStorage` 安全读取主题。
- 将主题写入 `localStorage`。
- 将主题应用到 `document.documentElement.dataset.theme`。

应用入口附近维护主题状态，并传给 `AppShell`。这样登录页、加载态和登录后页面都能通过根节点变量统一变更，不需要每个页面单独处理。

### AppShell 改动

`AppShell` 接收当前主题和切换回调，在顶部右侧 actions 中渲染图标按钮。按钮使用 `lucide-react` 的 `Moon` 和 `Sun`，复用现有 icon button 或 compact button 样式。

按钮不承载业务数据，不影响导航、语言选择、登录状态和退出流程。

### CSS 变量

在 `frontend/src/styles.css` 中保留现有 `:root` 浅色变量，并新增：

```css
:root[data-theme="dark"] {
  color-scheme: dark;
  --surface-page: #0b1220;
  --surface-card: #111827;
  --surface-muted: #162033;
  --surface-soft: #1f2937;
  --surface-elevated: rgb(15 23 42 / 94%);
  --border-subtle: #263447;
  --border-strong: #3a4a5f;
  --text-primary: #f8fafc;
  --text-secondary: #cbd5e1;
  --text-muted: #94a3b8;
}
```

实际实现时需要补齐状态色和阴影变量：

- `--accent-warm-soft` 在暗色中不能继续使用浅黄色底，应改为深暖色透明背景。
- `--success-soft`、`--danger-soft` 需要调成深色背景，避免状态 badge 过亮。
- `--shadow-subtle`、`--shadow-hover`、`--shadow-cta` 需要降低亮色阴影或改为暗色阴影。
- `--action-primary` 保留黄色，`--action-ink` 保持深色，保证主按钮对比度。

少量硬编码颜色需要一起处理。优先将与主题相关的硬编码背景、边框和文字颜色替换为已有变量；只在确实是固定品牌色或图形色时保留字面量。

### 国际化文案

主题按钮的可访问名称进入现有 i18n 资源：

- 中文：`切换为深色模式`、`切换为浅色模式`
- 英文：`Switch to dark mode`、`Switch to light mode`

不在页面中添加额外说明文案。

## 方案取舍

### 推荐方案：`data-theme` + CSS 变量覆盖

该方案利用现有样式基础设施，改动集中、风险低。所有页面通过同一套变量响应主题变化，后续新增页面只要继续使用变量即可自动支持暗色。

### 备选方案：逐组件添加 dark class

这种方式局部控制更细，但会把主题逻辑分散到多个组件和样式块，后续扩展和审查成本更高，不适合当前已有全局变量的代码形态。

### 备选方案：引入主题库或 CSS-in-JS

能力更完整，但会增加依赖、迁移成本和调试面。当前目标只是简单 light/dark 开关，不需要引入新的主题系统。

## 测试

前端测试覆盖以下行为：

- 没有本地存储主题时默认 `light`。
- 本地存储为 `dark` 时启动后应用暗色主题。
- 本地存储为非法值时回退 `light`。
- 点击顶部主题按钮后切换 `light` 和 `dark`，并写入 `localStorage`。
- 主题按钮在中文和英文环境下有正确可访问名称。
- 样式文件包含暗色变量定义，避免主题 token 被误删。

手动验证：

- 题库页的搜索框、筛选、列表、徽标、删除图标和空状态在暗色下可读。
- 学习计划列表、草案预览、聊天工作台和代码块在暗色下边界清晰。
- 登录页、认证加载态和错误态在暗色下没有白底残留。

## 风险与处理

- 部分旧样式仍可能使用硬编码浅色。实现时通过搜索 `#fff`、`white`、`rgb(255`、`rgb(15 23 42` 等主题相关颜色逐步收敛到变量。
- 黄色主按钮在暗色中会更醒目。第一版保留黄色动作色，只调整阴影和浅黄色背景变量。
- 浏览器或测试环境可能禁用 `localStorage`。主题工具需要捕获异常，保证应用不会因为存储失败崩溃。
- 主题切换如果只在 AppShell 内处理，登录页可能无法初始化主题。因此主题初始化需要放在应用入口或 App 顶层。

## 验收标准

- 打开应用默认是浅色。
- 登录后点击右上角月亮按钮，页面切换为中性深色，按钮变为太阳。
- 刷新页面后仍保持暗色。
- 再点击太阳按钮，页面恢复浅色并持久化。
- 登录页没有主题按钮，但会使用已保存主题。
- `make frontend-test` 通过，或至少相关 Vitest 用例通过。
